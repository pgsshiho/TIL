# 1. 휴리스틱에 대한 개념
**사용성에 대한 문제를 찾아내기 위한 사용성 공학 방법, 규칙들을 얼마나 잘 지키고 있는지를 확인 하는 방법**
# 2. 사용자 중심 디자인 개념
**사용자를 중심으로 사고하여 만들어 내는 인터페이스,서비스,제품디자인의 철학적 접근 방법이자 디자인 사상이다.**
# 3. 와이어 프레임의 개념
**최종 화면에 표시괼 콘텐츠를 간단히 요약하여 보여주는 것**
# 4. 맥락적 인터뷰 개념
**특정 상황이나 맥락에서 이루어지며,특정 행동을 관찰하고 조사할 수 있는 방법**
# 6. 효율성 매트릭스
**테스트를 완료하기 위해서 걸리는 시간 뿐 아니라, 인지적 노력과 물리적 노력의 양을 중요하게 여길 때 효과적**
# 8. 그래픽 사용자 인터페이스의 개념
컴퓨터를 사용하면서,그림으로 된 화면 위의 물체나 특성, 색상과 같은 그래픽 요소들을 어떠한 기능과 용도를 나타내기 위해 고안된 사용자를 위한 컴퓨터 인터페이스를 의미
# 9. 파일럿 테스트의 개념
컴퓨터 프로그램 등의 최신 기술을 개발하여, 실제 상황에서 실현하기 전에 소규모로 시험 작동해보는 것
# 10. 심층 인터뷰에
**1대1**면접을 통해 소비자의 심리를 파악하는 조사법, 응답자의 내면 깊숙이 자리잡고있는 **욕구,태도,감정** 등을 발견하는 면접 조사
# 11. 멘탈 모델
**사람들이대상 제품을 어떨게 사용하고 싶어하는지 이해하기 위해 사람들이 하는 행동들의 패턴을 찾고 그 패턴을 정의** 
# 12. UI 컨셉션의 개념
UX컨셉의 가시화 단계에서 진행되는 주요 화면에 대한 UI컨셉 설계를 의미한다.
# 13. 사용성 테스트 기법 선정하기 수행절차(가),(나),(다),(라)
(가):UI 테스트의 목적,필요성, 중요성에 대해서 이해관계자들 간 **인식 공유**  
(나): 테스트 대상의 계획,요구분석,콘셉트 기획 등 단계벌 세부 **내용 파악**  
(다): 다양한 UI 테스트 **기법에 대한 조사 및 이해**  
(라): 구현된 UI를 테스트하기에 가장 적절한 **기법을 선정**  
# 14. 수행절차(가),(나),(다)
1. **( 가 )**  
2. **테스트 항목 정의**  
3. **( 나 )**  
4. **테스트 룸 설정**  
5. **( 다 )**  
6. **테스트 인원 구성**  
( 가 ) :  테스트 목표 설정, ( 나 ) : 테스트 참여자 확보, ( 다 ) : 테스트 환경 설정
# 15. 태스크 성공 매트릭스의 개념
개선 이전의 UI/Ux 디자인과 개선 이후의 UI/Ux 디자인의 태스크 성공 여부(이진성공률)의 빈도 분포로 표현하여 비교
# 16. 카드소팅의 개념
정보구조를 알 수 있는 가장 단순하면서도 효과적인 방법 중의 하나로서,  
아이디어와 컨셉을 작은 카드에 적고 사용자가 카드를 그룹으로 분류 및 정렬하여 구조화시키는 방법
# 17. ISO/IEC 9126의 품질 보델의 제 4계층 구조를 서술
소프트웨어 특성을 측정하기 위한 메트릭이나 품질인자가 위치한다.
# 18. NEM 기법에 대한 설명이다. (가),(나),(다),(라)
대상 제품을 **처음으로 사용**해 보는 (가)와 대상 **제품의 설계 및 개발에 참여한** (나) 를 (다)에 참여시켜서  
대상 제품의 (라)를 **비교하여** 문제점을 객관적으로 제시하는 사용성 테스트 기법
(가): 초심자, (나): 숙련자 (다): 사용성 테스트, (라):테스트 수행시간
# 19. 정략적 리서치 개념
경쟁사 사이트, 벤치마킹사이트 또는 이전 디자인과 비교하여 현재 나의 UX 디자인 수준을 평가 하는 방법
# 20. 보기의 개념
기업이 참여하고 있는 사업 분야의 **시장 현황** 및 특성을 파악하여, 시장 매력 요인을 도출하고 성공가능성을 예측하기 위한 **보고서**이다.  
시장현황보고서
# 21. 포커스 그룹 인터뷰의 개념
표적시장으로 예상되는 소비자를 일정한 자격 기준에 따라 6~12명 정도 선발하여, 한 장소에 모이게 한 후,  
면접자의 진행 아래 조사 목적과 관련된 토론을 함으로써 자료를 수집하는 방법
# 22. 테스트 기법 선정 중 보기의 해당하는 내용
“A가 B보다 더 좋다”, “C가 D보다 더 편리하다”와 같이 제품이나 서비스에 대한 사용자의 **선호도**에 영향을 미치는 속성들을 파악하고,  
중요도에 따른 선호도를 예측하기 위하여 사용된다. 사용자의 니즈에 대응할 수 있는 **평가**방법이다.  
선호도 평가

# 26. 사용성 테스트 진행 인원별 역할
테스트 진행자 : 제품에 대한 중립적인 사람으로서 테스트 진행 및 결과의 책임자
기록 담당자 : 테스트 과정에서 발생하는 중요 이벤트를 기록하여 참여자의 의견을 기록
테스트 관찰자 : 대상 제품의 개발에 직간접적인 관련이 있는 사람
