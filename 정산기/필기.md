# 애플리케이견 배포
## 배포 환경
### 컴파일 언어
- 한번에 기계어로 변환
- 변역 시간이 오래걸림
- 번역후 프로그램 속도가 빠르고 성능 우수
#### Byte Code 언어
- 컴파일의 결과물이 실행파일이 아닌 'class'라는 바이트 코드로 파일을 생성하여 가상 실행환경인 JRE, CLR에서 한 줄씩 실행하는 방식이다.
#### 인터프리터 언어 
- 한 줄씩 기계어로 번역
- 메모리 적게 소모
- 컴파일 시간이 빠름
- 필요 할때만 번역해 프로그램 시간은 늦음
### 애플리케이션 배포(Release) 환경
개발자 또는 사용자가 애플리케이션을 실행, 테스트할 수 있도록 컴파일된 프로그램, 실행에 필요한 리소스(이미지,환경설정 파일 등)를 서버상의 적합한 위치로 이동하는 작업을 말한다.
1. 웹서버 
    사용자의 요청자료(정적인 데이터 : html, 그림, 동영상 등)의 결과값을 빠르고 안정적으로 처리하여 제공한다.

2. WAS(Web Application Server)
    사용자의 요청자료(동적인 데이터 : 연산,테이블 검색, 삽입, 삭제등 )의 결괏값을 빠르게 안정적으로 처리하여 제공한다.
### 애플리케이션 배포 단위
1. jar 
    java 라이브러리,리소스,property 파일들을 포함. 프로그램에서 참조하는 라이브러리, 구현된 비지니스 서비스를 배포한 때 jar 단위로 패키징하요 배포 (부품들의 집합)

2. war 
    웹 컨테이너에 배포되는 배포 형식으로 Servlet, jar 파일과 web-inf 폴더에 있는 web.xml 파일로 구성된다. 웹 컨테이너상에 배포되어 독립적인 UI 및 웹 애플리케이션 서비스를 제공할 수 있다.(부품으로 만든 하나의 가구)

3. ear 
    jar와 war을 묶어서 하나의 완성된 웹 애플리케이션 서비스를 제공할 수 있다.(가구들을 모아놓은 집)
### 형상관리
- 정의
    형상관리는 형상항목을 식별하여 그 기능적, 물리적 특성을 문서화하고, 그러한 특성의 변경, 제어, 처리 상태를 기록 및 보고하면서 명시된 요구사항에 부합하는지 확인 및 감독하는 활동을 의미한다.
- 요약
    Github에서 하던거들
## 애플리케아션 소스 검증하기
### 소스코드 검증도구
1. 정적 테스트 도구 
    테스트하기 전에 코딩오류, 성능저하, 보안 취약점 등의 결함을 조기에 발견할 수 있도록 지원한다. 이렇게 하여 프로그램 생산성 향상과 품질향상에 필요한 관리 시스템을 구축.
2. 동적 테스트 도구 
    테스트 미수행 코드를 확인 및 코드구조가 충분히 테스트 되었는지 확인하여 애플리케이션의 안정성을 높이기 위한 관리 시스템을 구축
- 요약
    정적 -> 코드작성시 빨간줄
    동적 -> 실행해서 오류나는거 확인
### 코드 인스팩션
- 정의
    정적 테스트의 가장 일반적인 유형으로, 사전에 정의된 코드 작성 규칙 기반으로 소스코드를 점검하여 작성 규칙에 위반되는 소스코드를 추출하여 점검
1. 코드 인스팩션 규칙 3가지
- 성능 개선
- 코드 작성 규칙
- 에러 발생 가능성
2. 정규 표현식
- 정의
    특정한 규칙을 가진 문자열의 집합을 표현하는 범용적인 방식을 말한다. 코드 인스팩션 도구의 코드 작성 규칙은 일반적으로 정규식으로 표현되며, 정규식의 내용을 수정해서 점검 규칙의 내용을 수정할 수 있다.
- 요약
    규칙을 작성할때 쓸 작성방식
### 테스트 프레임워크의 구성
- 정의
    테스트 케이스를 별도의 테스트 코드로 작성하고 동작시킬 수 있는 환경을 제공하는 도구로, 개발자의 반복적이고 시간이 많이 소요되는 테스트 작업을 자동화하여 테스트에 소요되는 시간과 노력을 절감할 수 있게 한다.

# 프로그래밍 언어 활용
## 프로그래밍 기본 문법
### 비트
- 정의
    컴퓨터를 이해하기 위한 가장 기본적인 용어로, Binary Digit의 약칭이다. 0과 1로만 구성된 이진법을 이용한다
    (bit -> byte -> KB -> MB -> GB -> TB -> PB -> EB)
### 컴퓨터 시스템 구조
 - 입력장치
 - 기억장치
 - 중앙처리장치
 - 출력장치
### 명령어 형식
 - 연산코드
    컴퓨터가 수행할 명령어를 말한다.
 - 오퍼랜드
    연산코드의 명령을 수행하기 위해 필요한 자료를 말한다.
### 자료구조
- 정의
    컴퓨터에 데이터를 삽입, 삭제, 수정하게 해 주는 논리적인 공간 구조를 의미하며, 자료의 형태에 따라 단순 구조, 선형 구조, 비선형 구조, 파일 구조로 분류할 수 있다.
- 단순 구조
    프로그래밍 언어에서 제공하는 기본 데이터 타입(int,float등)
- 선형 구조
    자료들 사이의 선후 관계가 일대일 구조 (연결 자료 구조,스택,쿠 등)
- 비선형 구조
    자료들 사이의 선후관계가 계층 또는 그불 형태를 가지는 구조(트리,그래프 등)
- 파일 구조
    보조 기억 장치에 데이터값이 실제로 기록되는 자료 구조(슨차 파일, 색인 파일등)
### 알고리즘
- 정의
    알고리즘은 넒은 의미에서 자료 구조와 함께 프로그램을 구성하는 요소를 의미하며, 좁은 의미에서는 어떤 문제에 대한 답을 찾는 해법을 의미한다.
- 입력
    외부로부터 입력되는 자료가 0개 이상이어야 한다.
- 출력
    출력되는 결과가 1개 이상이어야 한다.
- 명확성
    각 명령어의 의미가 명확하여야 한다.
- 유한성
    정해진 단계를 지나면 종료되어야 한다.
- 유효성
    모든 명령은 실행이 가능한 연산들이어야 한다.
### 기타 용어
- 변수
    어떤 값을 주기적 장치에 기억하기 위해서 사용하는 공간을 의미한다
- 식별자
    프로그램의 구성 요소를 구별하기 위한 기준으로, 변수명이 식별자에 속한다.
- 바인딩
    변수와 변수에 관련된 속성을 연결하는 과정으로, 정적 바인딩과 동적 바인딩으로 구분된다.
        정적 바인딩: 프로그램 실행 시간 전에 속성을 연결하는 방식
        동적 바인딩: 프로그램 실행 시간에 속성을 연결하는 방식
- 선언
    변수에 이름, 데이터 타입 등의 속성을 부여하는 작업으로,명시적 선언과 묵시적 선언으로 구분된다.
        명시적 선언: 선언문을 이용하여 변수 이름을 나열하고 속성을 부여하는 방식이다.
        묵시적 선언 : 별도의 선언문 없이 디폴트 규칙에 의해 속성이 부여되는 방식이다.
- 영역
    이름이 사용되는 범위를 의미, 정적 영역과 동적 영역으로 구분
        정적 영역: 변수를 찾을 때 구조에 기반하는 방식
        동적 영역: 변수를 찾을 때 구조보단 순서에 기반하는 방식
- 할당
    변수에 메모리 공간을 바인딩하는 작업을 의미
- 데이터 타입
    변수가 가질 수 있는 속성값의 길이 및 성질을 의미
- 연산자
    데이터 처리를 위해 연산을 표현하는 기호로 +,-등과 같은 연산자를 포함
- 명령문
    프로그램을 구성하는 문장으로,지시 사항을 처리하는 단위를 의미
### 연산자와 명령문 활용
#### 연산자
- 산술연산자 : +,-,*,/,%
- 시프트 연산자 : <<,>>
- 관계 연산자 : >, <, >=, <+, ==, !=
- 논리 연산자 : &&,||
- 비트 연산자 : &,|,^(XOR)
#### 명령문
    - 조건문 : IF,case/switch
    - 반복문 : while,for,do~while
#### 사용자 정의 자료형 활용
    1. 열거형
        연속적인 값이 들어가는 자료형으로 특정 값을 넣어 주지 않으면 1씩 늘어나기 떄문에 상수 배열이라고 부르기도 한다.
    2. 구조체
        멤버 변수를 사용하는 자료형으로 내부에 멤버 변수 자료형을 마음대로 선언할 수 있다.
    3. 공용체
        구조체와 거의 유사하나 범주가 더 크다고 볼 수 있으며, 구조체를 멤버로 사용할 수 있다.
#### 추상화와 상속
- 추상화
    복잡한 문제의 본질을 이해하기 위해 세부 사항은 배제하고, 중요한 부분을 중심으로 간략화 기법
- 상속성
    상위 수준 그룹의 모든 특성을 하위 수준 그룹이 이어받아 재사용 또는 확장성을 높이는 기법
