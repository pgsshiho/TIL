# 애플리케이견 배포
## 배포 환경
### 컴파일 언어
- 한번에 기계어로 변환
- 변역 시간이 오래걸림
- 번역후 프로그램 속도가 빠르고 성능 우수
#### Byte Code 언어
- 컴파일의 결과물이 실행파일이 아닌 'class'라는 바이트 코드로 파일을 생성하여 가상 실행환경인 JRE, CLR에서 한 줄씩 실행하는 방식이다.
#### 인터프리터 언어 
- 한 줄씩 기계어로 번역
- 메모리 적게 소모
- 컴파일 시간이 빠름
- 필요 할때만 번역해 프로그램 시간은 늦음
### 애플리케이션 배포(Release) 환경
개발자 또는 사용자가 애플리케이션을 실행, 테스트할 수 있도록 컴파일된 프로그램, 실행에 필요한 리소스(이미지,환경설정 파일 등)를 서버상의 적합한 위치로 이동하는 작업을 말한다.
1. 웹서버: 사용자의 요청자료(정적인 데이터 : html, 그림, 동영상 등)의 결과값을 빠르고 안정적으로 처리하여 제공한다.

2. WAS(Web Application Server): 사용자의 요청자료(동적인 데이터 : 연산,테이블 검색, 삽입, 삭제등 )의 결괏값을 빠르게 안정적으로 처리하여 제공한다.
### 애플리케이션 배포 단위
1. jar : java 라이브러리,리소스,property 파일들을 포함. 프로그램에서 참조하는 라이브러리, 구현된 비지니스 서비스를 배포한 때 jar 단위로 패키징하요 배포 (부품들의 집합)

2. war : 웹 컨테이너에 배포되는 배포 형식으로 Servlet, jar 파일과 web-inf 폴더에 있는 web.xml 파일로 구성된다. 웹 컨테이너상에 배포되어 독립적인 UI 및 웹 애플리케이션 서비스를 제공할 수 있다.(부품으로 만든 하나의 가구)

3. ear : jar와 war을 묶어서 하나의 완성된 웹 애플리케이션 서비스를 제공할 수 있다.(가구들을 모아놓은 집)
### 형상관리
- 정의
형상관리는 형상항목을 식별하여 그 기능적, 물리적 특성을 문서화하고, 그러한 특성의 변경, 제어, 처리 상태를 기록 및 보고하면서 명시된 요구사항에 부합하는지 확인 및 감독하는 활동을 의미한다.

## 애플리케아션 소스 검증하기
### 소스코드 검증도구
1. 정적 테스트 도구 : 테스트하기 전에 코딩오류, 성능저하, 보안 취약점 등의 결함을 조기에 발견할 수 있도록 지원한다. 이렇게 하여 프로그램 생산성 향상과 품질향상에 필요한 관리 시스템을 구축.

2. 동적 테스트 도구 : 테스트 미수행 코드를 확인 및 코드구조가 충분히 테스트 되었는지 확인하여 애플리케이션의 안정성을 높이기 위한 관리 시스템을 구축
- 요약
정적 -> 코드작성시 빨간줄
동적 -> 실행해서 오류나는거 확인
### 코드 인스팩션
- 정의
정적 테스트의 가장 일반적인 유형으로, 사전에 정의된 코드 작성 규칙 기반으로 소스코드를 점검하여 작성 규칙에 위반되는 소스코드를 추출하여 점검
1. 코드 인스팩션 규칙 3가지
- 성능 개선
- 코드 작성 규칙
- 에러 발생 가능성
2. 정규 표현식
- 정의
특정한 규칙을 가진 문자열의 집합을 표현하는 범용적인 방식을 말한다. 코드 인스팩션 도구의 코드 작성 규칙은 일반적으로 정규식으로 표현되며, 정규식의 내용을 수정해서 점검 규칙의 내용을 수정할 수 있다.
- 요약
규칙을 작성할때 쓸 작성방식
### 테스트 프레임워크의 구성
- 정의
테스트 케이스를 별도의 테스트 코드로 작성하고 동작시킬 수 있는 환경을 제공하는 도구로, 개발자의 반복적이고 시간이 많이 소요되는 테스트 작업을 자동화하여 테스트에 소요되는 시간과 노력을 절감할 수 있게 한다.

테스트 메크로
